# AI Financial Advisor

AI Financial Advisor is a full-stack personal finance assistant built with FastAPI, machine learning, and SQLite. It helps users estimate savings trajectory, assess financial health, track daily spending insights, and manage historical submissions through a UI and API.

## 🚀 What this project offers

- User authentication with register/login/logout and session cookie support
- Budget and savings prediction using trained ML model (`ml/model.py`) with fallback rule-based calculations
- Risk/health analytics: overspending risk, savings velocity, personalized guidance
- Persistent storage of advisory sessions in SQLite (`financial.db`) via `database.py`
- Web interface via static pages in `static/` (`index.html`, `login.html`) and chart rendering for trends
- CLI setup workflow that ensures dataset/model generation on first run

## 📁 Repository structure

- `main.py` - App startup, route registration, request validation, and ML model loading logic. binds FastAPI routes to business controllers.
- `advisor.py` - Core financial advice engine. Evaluates spending patterns, risk thresholds, health score, and recommendation text.
- `charts.py` - Chart data transformation and generation utilities used by the UI for bar/trend/pie chart output in JSON.
- `database.py` - Database schema creation and CRUD helpers for users + advisor history in SQLite; password hashing and session user lookup.
- `setup.py` - Initialization script run once to create dataset, train model, and ensure DB is ready.
- `ml/` - ML dataset and modeling utilities
    - `generate_dataset.py` - Builds synthetic dataset (`ml/financial_data.csv`) if absent.
    - `model.py` - Trains RandomForest model, evaluates metrics, and saves `ml/model.pkl`.
- `static/` - Frontend templates and static assets
    - `index.html` - Authenticated dashboard UI with forms and charts.
    - `login.html` - Login/register form page.

## ⚙️ Prerequisites

- Python 3.10+
- `pip` package manager
- Windows, Linux, or macOS

## 🛠️ Setup and first-run

1. Create and activate virtual environment

```powershell
cd ai-financial-advisor
python -m venv venv
.\venv\Scripts\Activate.ps1  # Windows PowerShell
# or source venv/bin/activate on macOS/Linux
pip install --upgrade pip
pip install -r requirements.txt
```

2. First-time initialization (required once)

```powershell
python setup.py
```

- ensures `ml/financial_data.csv` exists
- trains model and writes `ml/model.pkl`
- creates `financial.db` and required tables

3. Run app

```powershell
uvicorn main:app --host 127.0.0.1 --port 8001
```

4. Optional development run with hot reload

```powershell
uvicorn main:app --host 127.0.0.1 --port 8001 --reload --reload-exclude "venv/*"
```

## 🧩 API endpoints

### Authentication
- `POST /auth/register` - JSON: `{ "username": "...", "password": "...", "full_name": "..." }`
- `POST /auth/login` - JSON: `{ "username": "...", "password": "..." }` sets session cookie
- `POST /auth/logout` - clears session cookie

### Financial prediction and advice
- `POST /predict` - calculate expected monthly savings from input
- `POST /analyze` - compute financial health score + advice categories
- `POST /advisor` - log session and produce personalized guidance
- `GET /history` - retrieve latest advisory records for logged-in user

### Public pages
- `GET /` - dashboard (requires auth)
- `GET /login` - login page

### Request body model

`FinancialInput` contains:
- `income` (number)
- `fixed_expenses` (number)
- `variable_expenses` (number)
- `savings_goal` (number)
- `lifestyle_score` (number, typically 0-10)

## 🧪 Data and model details

- Synthetic dataset: `ml/financial_data.csv` generated by `ml/generate_dataset.py`
- Model training: `ml/model.py` builds a RandomForest regression model to predict savings.
- Serialization: model persisted to `ml/model.pkl`; loaded in `main.py` at startup.
- Fallback: missing/invalid `model.pkl` triggers retraining automatically.

## 💡 Implementation highlights

- Security: hashed passwords stored in DB (`database.py` handles hash check)
- Persistence: session data includes timestamps and user-specific records
- Analytics: `advisor.py` computes risk categories, spending benchmarks, and written guidance
- UI: front-end charts through `charts.py` endpoints + static HTML forms

## 🔍 Development and testing

- To inspect or retrain dataset/model manually:

```powershell
python ml/generate_dataset.py
python ml/model.py
```

- To reset database

```powershell
del financial.db    # Windows
rm financial.db     # Linux/macOS
python setup.py
```

## 🧾 Troubleshooting

- `ModuleNotFoundError`: install from `requirements.txt` and re-activate venv
- `ValueError` in event loop reload: normal behavior with uvicorn `--reload`
- `PermissionError` on `financial.db`: ensure file not locked and app has write permission

## 📌 Notes

- Designed for local demo / prototype; not hardened for production use without additional security, CSRF protection, input validation, hosting config, and HTTPS.
- Compatible with Windows; recommended UTF-8 terminal support for improved logging
